# システム概要・ドメイン構成

## 目的

プロジェクト管理を中心に、組織・人員・文書を一元的に追跡するシステムを構築する。
文書管理はその基盤インフラとして位置づけ、**どの組織が・どのプロジェクトで・何の文書を・誰が作ったか**を追跡できるようにする。

文書本体（バイナリファイル）はローカルまたは NAS に保存し、本システムは**パスと属性メタデータのみ**を管理する。

## 背景

- 設備保全・機械電気工事を扱う技術部門を対象とする
- 各プロジェクトは SAP の WBS で管理されているが、文書メタデータの検索・承認には対応していない
- 部署は再編・統廃合があるため、文書の帰属先を変更なく追跡できる仕組みが必要
- 部署・社員情報は Active Directory（AD）を正とし、取得・加工してこのシステムへ同期する

## ドメイン一覧

| ドメイン     | 説明                                                                                |
| ------------ | ----------------------------------------------------------------------------------- |
| 部署         | AD の OU 階層を取得・加工して同期する組織単位。有効期間モデルで廃止・統合を管理する |
| 社員         | AD のユーザー情報を取得・加工して同期する人員マスタ。退職後もレコードを保持する     |
| 社員所属履歴 | 社員が所属した部署を期間付きで管理する中間テーブル                                  |
| 業務種別     | CAD・FEM・機械補修・電気工事などの技術領域マスタ。担当部署を持つ                    |
| プロジェクト | WBS コードを参照情報として保持する独自エンティティ。業務種別を経由して部署と紐づく  |
| 文書種別     | 契約・議事録・社内文書などの文書の性質による分類。採番の連番桁数を持つ              |
| 文書台帳     | 文書種別と部署の組み合わせごとに採番ルール・保管先・命名規則を定義する              |
| 文書         | ファイルパス・メタデータ・機密グレードを保持する。必ずプロジェクトに紐づく          |
| タグ         | 文書に付与する自由分類ラベル                                                        |
| 承認ルート   | 文書の段階承認を順序付きで管理する                                                  |
| 配布         | 文書を社員にメール配布し、配布履歴を管理する                                        |

## ドメイン間の依存関係

```text
部署
 ├─ 社員所属履歴 ──── 社員
 ├─ 業務種別
 │    └─ プロジェクト
 │         └─ 文書（project_id, author_id）
 │              ├─ タグ（中間テーブル）
 │              ├─ 承認ルート
 │              └─ 配布
 └─ 文書台帳 ──── 文書種別
```

プロジェクトは部署を直接持たない。業務種別を経由して間接的に部署と紐づく。

## スコープ外

以下は本システムの対象外とし、既存システムに委ねる。

| 機能               | 委譲先                                   |
| ------------------ | ---------------------------------------- |
| タスク・工数管理   | Redmine / Jira                           |
| 予算・原価管理     | SAP                                      |
| 文書バージョン管理 | ファイルサーバー（Git / NAS の履歴機能） |
| 契約・請求管理     | SAP                                      |

## 権限モデル

RBAC（Role-Based Access Control）で 4 段階のロールを定義する。

| ロール            | 説明                                               |
| ----------------- | -------------------------------------------------- |
| `admin`           | 全操作。部署・社員マスタの管理を含む               |
| `project_manager` | 担当プロジェクト・文書の登録・更新・承認ルート設定 |
| `general`         | 文書登録・参照（機密グレードの範囲内）             |
| `viewer`          | 参照のみ（機密グレードの範囲内）                   |

## 機密グレード

文書に設定する固定 4 段階。

| 値             | 説明   |
| -------------- | ------ |
| `public`       | 公開   |
| `internal`     | 社内   |
| `restricted`   | 部外秘 |
| `confidential` | 機密   |
