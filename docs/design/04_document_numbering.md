# 文書番号採番仕様

## フォーマット

```text
{種別コード}{部署コード}-{YYMM}{連番}
```

| フィールド | 例     | 説明                                                                     |
| ---------- | ------ | ------------------------------------------------------------------------ |
| 種別コード | `CAD`  | `business_categories.code`。最大 10 文字                                 |
| 部署コード | `001`  | 業務種別の担当部署コード（`departments.code`）。最大 10 文字             |
| `-`        | `-`    | 固定区切り文字                                                           |
| YYMM       | `2602` | 西暦年下 2 桁 + 月 2 桁（登録年月）                                      |
| 連番       | `001`  | 同一 `{種別コード}{部署コード}{YYMM}` 内の通し番号（3 桁以上、ゼロ埋め） |

### 例

```text
CAD001-2602001   # CAD部門001番、2026年2月、1件目
CAD001-2602002   # CAD部門001番、2026年2月、2件目
FEM002-2603001   # FEM部門002番、2026年3月、1件目
```

---

## 採番手順

文書登録リクエストを受けた際、以下の順序で採番する。

1. リクエストから `project_id` を取得する
2. `projects` テーブルから `business_category_id` を取得する
3. `business_categories` テーブルから `code`（種別コード）と `department_id` を取得する
4. `departments` テーブルから `code`（部署コード）を取得する
5. 登録日時から `YYMM` を生成する（サーバー時刻の UTC → JST 変換後の年月）
6. `documents` テーブルで `frozen_dept_code = {部署コード}` かつ `doc_number LIKE '{種別コード}{部署コード}-{YYMM}%'` となる最大連番を取得する
7. 連番をインクリメントし、3 桁以上にゼロ埋めして文書番号を生成する
8. 生成した文書番号を `doc_number` に、部署コードを `frozen_dept_code` にセットして保存する

### 採番のアトミック性

同一の `{種別コード}{部署コード}{YYMM}` に対して同時採番が発生した場合に重複しないよう、採番処理全体をトランザクション内で実行し、`documents` テーブルに対して `SELECT ... FOR UPDATE` または `SERIALIZABLE` 分離レベルを使用する。

---

## 凍結ルール

採番後の `doc_number` および `frozen_dept_code` は**変更不可**とする。

- `PUT /api/v1/documents/:id` では `doc_number`, `frozen_dept_code` を更新対象から除外する
- アプリケーション層でこれらのフィールドを無視し、DB に反映しない

---

## 部署再編時の挙動

### 部署廃止・統合

- 廃止部署に紐づく業務種別の `department_id` を新しい担当部署に更新する
- 廃止部署のコード（`departments.code`）は**再利用しない**
- 過去に採番済みの文書番号は変更しない（`frozen_dept_code` が凍結済みのため）
- 廃止部署の有効期間を `effective_to = 廃止日` に更新する

### 採番への影響

| タイミング           | 挙動                                               |
| -------------------- | -------------------------------------------------- |
| 廃止前に登録した文書 | 旧部署コードが `frozen_dept_code` に残る。変更なし |
| 廃止後に登録した文書 | 新担当部署のコードで採番される                     |

### 連番の継続性

部署コードが変わった場合（例: `001` → `010`）、連番は新しい部署コードで 1 から始まる。
旧コードの連番と混在することはない。

---

## 種別コードの管理

`business_categories.code` は業務種別マスタに登録する際に決定する。
一度登録したコードは変更しない。

現在想定している種別コード例:

| コード | 業務種別名 |
| ------ | ---------- |
| `CAD`  | CAD 設計   |
| `FEM`  | FEM 解析   |
| `MECH` | 機械補修   |
| `ELEC` | 電気工事   |
