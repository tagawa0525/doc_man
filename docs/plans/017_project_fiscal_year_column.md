# プロジェクト一覧に年度カラムを追加し、ソート順を変更する

## Context

プロジェクト管理の一覧テーブルに「年度」が表示されておらず、どの年度の案件か一目でわからない。
また、デフォルトのソート順を「新しい年度が上、年度内では古い案件が下」に変更する。

## 方針

**フロントエンドのみの変更**。APIは既に `start_date` を返しており、`fiscal_year_of()` 関数も存在するため、サーバー側の変更は不要。

修正対象: `frontend/src/pages/projects/list.rs`（1ファイルのみ）

## 変更内容

### 1. 年度カラムの追加

テーブルの先頭列（WBSコードの左）に「年度」カラムを追加する。

カラム順: **年度** | WBSコード | 名前 | マネージャー | ステータス | 文書

- 検索テーブル（L398-416）: ヘッダー行に `<th>"年度"</th>` 追加、検索入力行に空の `<th>` 追加（年度のフィルタリングは既存のチェックボックスで対応済み）
- データテーブルヘッダー（L474付近）: ソート可能な `<th>` を追加（キー: `"fiscal_year"`、初期ソート方向: DESC）
- データテーブルボディ（L589付近）: `<td>` を追加。`start_date` から `fiscal_year_of()` で算出。`start_date` が None の場合は空文字表示

表示形式: 4桁の西暦（例: "2025"）。既存の年度フィルタチェックボックスと同じ形式。

### 2. ソートロジックの変更

- `Some("fiscal_year")` マッチアームを追加（既存パターンに準拠）
- デフォルトソート（`_ =>` アーム、L456-467）の二次ソートを変更:
  - 変更前: `a.start_date.cmp(&b.start_date)` （start_date ASC = 古い案件が上）
  - 変更後: `b.start_date.cmp(&a.start_date)` （start_date DESC = 古い案件が下）

結果: 年度 DESC → 同一年度内は start_date DESC

### 3. 既存の分野グルーピングはそのまま維持

## 検証

1. `just fmt` / `just lint` でフォーマット・リントチェック
2. `just frontend-dev` で動作確認
   - 年度カラムが表示されること
   - 年度ヘッダークリックでソート切替が動作すること
   - デフォルト状態で新しい年度が上、年度内で古い案件が下にソートされること
   - start_date が null のプロジェクトが正しく表示されること
